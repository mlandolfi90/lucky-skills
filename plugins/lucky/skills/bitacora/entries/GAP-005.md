## [GAP-005] Docker Desktop no arranca: cartel "An unexpected error occurred" y el backend crashea en loop

- **TIPO:** GAP
- **SÍNTOMA (lo observable, NO la causa):** Docker Desktop muestra "An unexpected error
  occurred… needs to close" al arrancar; el error del recuadro nombra un socket bajo
  `AppData` (`…\Docker\run\dockerInference` o `…\docker-secrets-engine\engine.sock`) con
  "The file cannot be accessed by the system"; `docker info` → pipe not found; la distro
  WSL `docker-desktop` queda Stopped. Reintentar arrancar reproduce el cartel.
- **CAUSA-RAÍZ (1 línea):** bug de Docker Desktop 4.7x: tras un cierre sucio, los sockets
  unix quedan en estado inborrable y el backend crashea entero en vez de ignorarlos — y cada
  crash es un cierre sucio que siembra el siguiente (loop). `EnableDockerAI: false` NO lo evita.
- **ACCIÓN (pasos, máx 7, copy-paste si aplica):**
  1. Matar procesos: `Get-Process *docker* | Stop-Process -Force` + `wsl --shutdown`.
  2. Renombrar (borrar suele fallar): `%LOCALAPPDATA%\Docker\run` → `run.bak.<n>` y
     `%LOCALAPPDATA%\docker-secrets-engine` → `.bak.<n>`.
  3. Relanzar Docker Desktop; verificar `docker info`.
  4. Fix definitivo: actualizar a ≥4.80 (si winget cuelga en Delivery Optimization, bajar el
     instalador directo de desktop.docker.com y validar `Get-AuthenticodeSignature` = Valid).
- **ANTI-ACCIÓN (el camino muerto):** "Reset to factory defaults" del cartel (borra imágenes y
  volúmenes); reintentar el arranque en loop; borrar el socket con `Remove-Item`/`del` (falla:
  renombrar el directorio padre sí funciona).
- **PREVENCIÓN (cómo evitar reincidencia):** actualizar Docker Desktop; evitar force-kill del
  backend cuando alcanza con Quit; si reaparece el cartel, leer QUÉ ruta nombra antes de cerrar.
